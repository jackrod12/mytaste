## 1. 앱 컨셉

### 핵심 문제
주류 구매 현장에서 두 가지 문제가 동시에 발생한다.

1. **정보 비대칭** — 술에 대한 지식이 없으면 판매원에게 묻거나 직접 검색해야 함
2. **개인화 부재** — 검색으로 얻은 정보가 "내 입맛"에 맞는지 알 수 없음

### 핵심 솔루션
> "이 술, 나한테 맞아?"

마신 술을 기록하면 나만의 취향 프로필이 만들어지고, 새로운 술을 접했을 때 **객관적 정보 + 나에게 얼마나 적합한지**를 함께 알려주는 앱.

### 데이터 아키텍처
```
[글로벌 주류 DB]              [유저 개인 DB]
 술의 객관적 정보               내 음용 기록 + 평가
 (향, 맛, 계열, 도수 등)        (별점, 태그, 테이스팅 팔레트)
         ↘                        ↙
                [AI 분석 레이어]
           객관적 정보 + 개인 취향 매칭
```

## 3. MVP 카테고리 및 테이스팅 팔레트

### 지원 카테고리 (Phase 1)
와인 / 위스키 / 맥주 / 사케 / 막걸리 / 기타

### 서브카테고리 분류 체계
각 카테고리 내 세부 분류(품종·스타일·지역 등)는 추후 확정 예정.
유저의 탐색 기능("내가 마셔본 범주")과 취향 분석의 정밀도를 높이기 위해 필요하며, 실제 데이터가 쌓이는 시점에 전문 자료를 참고해 정의한다.

### 사용자 레벨
- **일반 유저** — 별점(1~10) + 태그 선택 (빠르고 간편)
- **고급 유저** — 테이스팅 팔레트 슬라이더 (정밀한 취향 분석)

### 카테고리별 테이스팅 팔레트

팔레트 항목은 DB 테이블(`palette_definitions`)로 관리. 코드 수정 없이 카테고리·서브카테고리 추가 가능.

**팔레트 시각화**: 항목 수에 따라 자동으로 다각형 결정 (5항목=오각형, 6항목=육각형 등). 각 항목 값(1~5)을 꼭짓점으로 레이더 차트 렌더링. 예시: 산도3 당도2 바디감1 타닌2 오크3 과실향4 → 육각형.

**와인 (기본)**
- 기본(6): 산도, 당도, 바디감, 타닌, 오크, 과실향
- 고급(+6): 꽃향, 미네랄, 향신료, 피니시 길이, 피니시 복잡도, 탁도
- 스파클링 서브카테고리 한정 추가: 탄산감 (palette_definitions의 sub_category 조건으로 처리)

**위스키**
- 기본(5): 피트·스모키, 달콤함, 스파이시, 바디감, 피니시
- 고급(+5): 과실향, 오크, 알코올 자극, 피니시 길이, 피니시 복잡도

**맥주**
- 기본(5): 쓴맛(IBU감), 탄산감, 단맛, 바디감, 홉향
- 고급(+5): 몰트향, 효모향, 산도, 로스팅, 피니시

**사케**
- 기본(5): 단맛, 산도, 바디감, 감칠맛(우마미), 향 강도
- 고급(+4): 쌀향, 발효향, 청량감, 피니시

**막걸리** (Phase 2 추가 예정)
- 기본(5): 단맛, 산도, 탄산감, 바디감, 쌀향
- 고급(+3): 발효향, 탁도, 피니시

## 4. 핵심 기능

### 4-1. 대시보드
- 나의 통계 (총 기록 수, 접속 횟수, 위시리스트 수)
- 카테고리별 취향 프로필 탭 (와인·위스키·맥주 각각 다른 태그·설명)
- 카테고리별 경험 횟수 + 비율 바
- 이번 주 추천 제품 (날씨·기온·시기 + 취향 기반 + 브랜드 이벤트)
  - 추천 이유를 3레이어로 표시: 날씨 / 취향 매칭 / 이벤트
  - 추후 광고 구좌로 전환 가능
- 액션 버튼: 기록하기 / 위시리스트 / 검색

### 4-2. 기록하기 플로우
1. **카테고리 선택** — 6개 카테고리 중 선택
2. **제품 검색** — 한글 기본, 언어 전환 가능 / 라벨 스캔 (Google Vision API)
   - 와인: 빈티지 연도 선택 (2026~1970, 알 수 없음 포함)
   - 검색 결과 없으면 "직접 추가하기" 옵션
3. **제품 상세** — 한글 제품명 + 영문명 / 생산자 / 지역 / 나라 / 풍미 태그 / 특성 태그 / 외부 평가 태그 / 테이스팅 팔레트 레이더 차트 / Vivino 점수 + 평가자 수 / 자세히 알아보기 (품종·지역·생산자 아코디언) / 비슷한 제품 추천
4. **테이스팅 노트** — 기본(별점+태그) or 고급(팔레트 슬라이더) 선택
5. **음용 기록** — 날짜 / 장소 / 함께 먹은 음식 / 누구와 / 한줄평(공개)
6. **완료** — AI 취향 분석 업데이트 코멘트 표시

### 4-3. 위시리스트 플로우
- 저장된 술 목록: 한글 제품명 + 영문명 + 풍미 태그 + AI 적합도 점수
- 적합도 색상: 90+ 초록 / 70~89 주황 / 70 미만 빨강
- 점수별 분석 구조 (합계 5~6가지, 긍정·걱정 비율은 점수에 따라 조정):
  - **90점 이상** — 긍정 요인 4~5가지 + 걱정 요인 0~1가지
  - **70~89점** — 긍정 요인 2~3가지 + 걱정 요인 2~3가지
  - **70점 미만** — 긍정 요인 1~2가지 + 걱정 요인 3~4가지
- 음식 페어링 4가지
- 비슷한 대안 가로 스크롤
- "마셨어요 — 기록하기" 버튼으로 기록 플로우 연결

### 4-4. 검색
- 텍스트: 한글 기본, 언어 전환 가능
- 라벨 스캔: 이미지 인식 후 이름·빈티지 유저 수정 가능
- DB 있는 술: 제품 정보 + 취향 적합도 표시 (기록 5개 이상 유저만)
- DB 없는 술: AI 웹서칭 → pending 저장 → 승인 후 등록. 유저에게 "잠시 후 등록됩니다" 안내
- 필터: 카테고리 / 나라 / 도수 범위

### 4-5. 소셜 기능 (Phase 2 이후)
- 한줄평 공개 피드 (유저가 동의한 것만)
- 취향 프로필 공유 카드 (SNS 바이럴 유도)

## 6. AI 호출 설계

### 호출 시점 4가지

| 시점 | 트리거 | 내용 | 빈도 |
|------|--------|------|------|
| 신규 술 정보 수집 | 유저가 "AI로 찾아보기" 클릭 | 웹서칭 → pending 저장 | 건당 1회, 이후 재호출 없음 |
| 위시리스트 적합도 분석 | 위시리스트 추가 시 | 글로벌 DB + 취향 프로필 → 점수·이유·페어링 | 추가 시 1회, 프로필 업데이트 시 재계산 |
| 취향 프로필 업데이트 | 기록 5개 추가마다 자동 | 전체 기록 분석 → 카테고리별 요약·태그 재생성 | 5회 기록마다 1회 |
| 주간 추천 생성 | 주 1회 자동 (월요일 오전) | 취향 프로필 + 날씨 API + 시즌 → 추천 1개 | 주 1회 |

### 캐싱 원칙
- 글로벌 주류 정보는 AI가 한 번만 수집. 두 번째 유저가 같은 술 검색 시 저장 데이터 재사용
- 적합도 분석은 유저마다 다르므로 wishlist 테이블에 유저별 저장
- 취향 프로필은 기록 5개마다 재계산하여 user_taste_profile에 캐싱

### 비용 시뮬레이션 (MAU 100명 기준)
- 신규 술 수집: 월 300회
- 위시리스트 분석: 월 200회
- 취향 프로필 업데이트: 월 200회
- 주간 추천: 월 400회
- **합계: 약 1,100회 × $0.003 ≈ 월 $3.3**

## 10. 롤백 전략

### 코드
- Git 브랜치 전략: main(프로덕션) / develop / feature 브랜치
- 기능 단위로 작게 쪼개서 배포. PR 단위로 리뷰 후 머지
- **기능 플래그**: 새 기능은 플래그로 숨겨두고 배포 없이 켜고 끌 수 있게

### 데이터베이스
- Supabase migration 파일을 Git에 버전 관리
- 마이그레이션 작성 시 rollback 스크립트 반드시 함께 작성
- Supabase point-in-time recovery 활성화 (특정 시점으로 DB 복원 가능)

### 앱 배포
- Expo EAS Update로 JS 변경은 앱스토어 심사 없이 OTA 즉시 배포
- 문제 발생 시 이전 번들로 즉시 롤백 (다운타임 없음)
- 네이티브 코드 변경이 필요한 경우에만 앱스토어 심사 경유

### 스테이징 환경
- 프로덕션과 동일한 환경을 별도 구성
- 모든 배포는 스테이징 → 프로덕션 순서로 진행
